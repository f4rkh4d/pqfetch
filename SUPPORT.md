# support

a few channels, in order of how fast you'll get a response:

## bugs

if it crashes, gives a wrong result, or doesn't build on a stable
toolchain: please open an issue at
https://github.com/f4rkh4d/pqfetch/issues with the steps to reproduce.

## questions

usage and design questions go in
[github discussions](https://github.com/f4rkh4d/pqfetch/discussions).
i answer the q&a category roughly daily.

## security

email **hello@frkhd.com** with subject `pqfetch security`. coordinated
disclosure preferred. see [SECURITY.md](SECURITY.md) where one ships
in this repo.

## not supported

- private email for general support: i don't have the throughput.
- discord / slack / matrix: i'm not in any of those.
- "can you build me a custom feature": open an issue describing the
  use case, i'll triage; if it's a fit i'll merge a PR, if not i'll
  say so.
